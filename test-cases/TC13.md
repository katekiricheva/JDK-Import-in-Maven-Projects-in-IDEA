# Test Case TC13 – Multi-module Java version via `toolchains.xml`

## Description
To verify that IntelliJ IDEA correctly imports Java language level when the version is set in the `toolchains.xml`.

## Prerequisites
1. Multi-module Maven project is freshly imported and opened in IntelliJ IDEA.
2. Toolchain is set in the parent `pom.xml` via `<maven-toolchains-plugin>`.
3. `toolchains.xml` is configured.

## Steps
1. Open the parent `pom.xml` file.
2. Verify that the toolchain is correctly set in the `<maven-toolchains-plugin>` section.
3. Open the `toolchains.xml` file.
4. Verify that the Java version is correctly defined there.
5. Navigate to: File → Project Structure → Project.
6. Check the "SDK" and "Language level" values.
7. Navigate to: File → Project Structure → Modules → Dependencies.
8. Verify the "Module SDK" settings for each module.
9. Navigate to: File → Project Structure → Modules → Sources.
10. Check the "Language level" settings for each module.

## Expected result
1. Project SDK matches the Java version specified in the `toolchains.xml`.
2. Module SDK is set to “Project SDK” for all modules.
3. Project language level matches the Java version specified in the `toolchains.xml` (not set to "SDK default").
4. Module language levels for all modules match the Java version specified in the `toolchains.xml` (not set to "Project default").
