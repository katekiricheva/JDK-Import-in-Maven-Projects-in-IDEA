# Test Case TC08 – Multi-module Java version inheritance via `<plugin>` `<release>`

## Description
To verify that IntelliJ IDEA correctly imports Java language level when the version is set via `<plugin>` `<release>` in the parent `pom.xml` and inherited by child modules.

## Prerequisites
1. Multi-module Maven project is freshly imported and opened in IntelliJ IDEA.
2. No additional configuration affects the Java version defined in the parent `pom.xml`.

## Steps
1. Open the parent `pom.xml` file.
2. Verify that the Java version is correctly defined in the `<plugin>` `<release>` section.
3. Navigate to: File → Project Structure → Project.
4. Check the "SDK" and "Language level" values.
5. Navigate to: File → Project Structure → Modules → Dependencies.
8. Verify the "Module SDK" settings for each module.
6. Navigate to: File → Project Structure → Modules → Sources.
7. Check the "Language level" settings for each module.

## Expected result
1. Project SDK matches the Java version specified in the parent `pom.xml`.
2. Module SDK is set to “Project SDK” with the Java version specified in the parent `pom.xml` for all modules.
3. Project language level matches the Java version specified in the parent `pom.xml` (not set to "SDK default").
4. Module language levels for all modules match the Java version specified in the parent `pom.xml` (not set to "Project default").
