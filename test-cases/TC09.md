# Test Case TC09 – Multi-module Java version override: parent via `<properties>` `<maven.compiler.source>` and `<maven.compiler.target>`, child via `<properties>` `<maven.compiler.release>`

## Description
To verify that IntelliJ IDEA correctly imports Java language level when the version set in the `pom.xml` of a child module via `<properties>` `<maven.compiler.release>` overrides the version set in the parent `pom.xml` via `<properties>` `<maven.compiler.source>` and `<maven.compiler.target>`.

## Prerequisites
1. Multi-module Maven project is freshly imported and opened in IntelliJ IDEA.
2. Java version is defined in the parent `pom.xml`.
3. One or more child modules set their own Java version in the `pom.xml`, overriding the version specified in the parent `pom.xml`.

## Steps
1. Open the parent `pom.xml` file.
2. Verify that the Java version is correctly defined in the `<properties>` `<maven.compiler.source>` and `<maven.compiler.target>` sections.
3. Open a child module’s `pom.xml`.
4. Verify the Java version set in the `<properties>` `<maven.compiler.release>` section.
5. Navigate to: File → Project Structure → Project.
6. Check the "SDK" and "Language level" values.
7. Navigate to: File → Project Structure → Modules → Dependencies.
8. Verify the "Module SDK" settings for each module.
9. Navigate to: File → Project Structure → Modules → Sources.
10. Check the "Language level" settings for each module.

## Expected result
1. Project SDK matches the Java version specified in the parent `pom.xml`.
3. Project language level matches the Java version specified in the parent `pom.xml` (not set to "SDK default").
4. Module SDK matches the Java version specified in the module’s `pom.xml` for all modules (not set to "Project SDK" for child modules).
5. Module language level matches the Java version specified in the module’s `pom.xml` for all modules (not set to "Project default").
